# 09 — Build Phases

Exact build order. Do not skip steps. Later phases depend on earlier ones being solid.

---

## Phase 1 — Foundation (Weeks 1–6)
**Goal: A working, shippable product you can use yourself every day.**

At the end of Phase 1, you have:
- Onboarding agent that learns who you are
- Morning + evening check-in loop
- Focus agent building a realistic daily plan
- Execution agent writing emails/docs on demand
- Chaos triage agent for overwhelm
- Basic dashboard showing today's plan, task queue, agent status
- Memory that persists between sessions

### Week 1: Infrastructure
- [ ] Set up repository structure (backend + frontend folders)
- [ ] Create `.env` file with all required keys (see `03-COSTS.md`)
- [ ] Run Docker Compose to start PostgreSQL + Redis + Chroma
- [ ] Install all Python dependencies (`requirements.txt`)
- [ ] Install all Node dependencies (`package.json`)
- [ ] Run Alembic migrations to create all tables
- [ ] Verify database connects: `python -c "from backend.db import engine; print('DB OK')"`
- [ ] Test Mem0 self-hosted with Chroma: add one memory, retrieve it, confirm it works
- [ ] Test Anthropic API: send one message to claude-haiku, confirm response
- [ ] Set up basic FastAPI app with one health check endpoint: `GET /health → {"status": "ok"}`

### Week 2: Memory Layer + User Auth
- [ ] Implement `POST /auth/register` — create user, hash password, return JWT
- [ ] Implement `POST /auth/login` — verify password, return JWT
- [ ] Implement JWT middleware — all protected routes require valid token
- [ ] Implement `save_memory(user_id, content, metadata)` helper function
- [ ] Implement `retrieve_memories(user_id, query, limit=10)` helper function
- [ ] Implement `get_user_profile(user_id)` from PostgreSQL
- [ ] Write tests for auth and memory functions
- [ ] Verify memory round-trip: save a fact, retrieve it with a related query

### Week 3: Supervisor + Onboarding Agent
- [ ] Create LangGraph state object (`LifeOSState`)
- [ ] Implement Supervisor Agent node (routing only, no content generation)
- [ ] Implement Onboarding Agent with 10-question conversational interview
- [ ] Implement `POST /onboarding/start` — triggers onboarding agent for new users
- [ ] Implement `POST /onboarding/message` — continues onboarding conversation
- [ ] After onboarding completes: save all extracted facts to Mem0, save structured data to `user_profiles`
- [ ] Test: complete a full onboarding interview, verify memories are saved correctly
- [ ] Implement `GET /onboarding/status` — returns whether onboarding is complete

### Week 4: Core Domain Agents
- [ ] Implement Focus Agent with `get_tasks`, `get_energy_pattern` tools
- [ ] Implement Health Agent with `log_health_entry`, `get_health_history` tools
- [ ] Implement Execution Agent with Tavily search tool
- [ ] Implement Chaos Triage Agent (no tools, reasoning only)
- [ ] Implement Synthesis Agent (reasoning only, runs last)
- [ ] Wire all agents into the LangGraph graph with supervisor routing
- [ ] Implement `POST /chat` endpoint — takes user message, runs agent graph, streams response via SSE
- [ ] Test each agent in isolation with mock data
- [ ] Test full supervisor → domain agent → synthesis flow end-to-end

### Week 5: Check-in System + Task Management
- [ ] Implement `POST /checkin/morning` — triggers morning check-in flow
- [ ] Implement `POST /checkin/evening` — triggers evening check-in flow
- [ ] Implement `POST /tasks` — create task
- [ ] Implement `GET /tasks` — list tasks for today
- [ ] Implement `PATCH /tasks/{id}` — update task (complete, defer, edit)
- [ ] Implement `DELETE /tasks/{id}` — delete task
- [ ] Implement APScheduler: send morning check-in reminder at 08:00 (user's local time)
- [ ] Verify: complete a morning check-in, see daily plan generated by Focus Agent

### Week 6: Frontend Dashboard
- [ ] Create Next.js app with authentication (login/register pages)
- [ ] Implement SSE consumer hook — connects to `/chat` SSE stream, displays tokens as they arrive
- [ ] Build Today page: daily plan (from Focus Agent), task queue, check-in card
- [ ] Build Chat page: talk to any agent, see which agent is responding
- [ ] Build Onboarding page: wraps the onboarding agent conversation
- [ ] Connect all frontend pages to backend API
- [ ] Deploy frontend to Vercel (free)
- [ ] Deploy backend to Railway or Fly.io
- [ ] **Phase 1 complete: use it yourself for 2 weeks before building Phase 2**

## Phase 2 — Pattern Intelligence (Weeks 7–12)
**Goal: The system learns your patterns and becomes genuinely smarter over time.**

At the end of Phase 2, you have:
- Nightly pattern learning job running automatically
- Time realism agent calibrated to your actual pace
- Energy rhythm model built from 2+ weeks of check-in data
- Cross-domain insights surfacing in weekly review
- Google Calendar integration for passive scheduling data
- Goals and relationships agents

### Week 7: Pattern Learning Agent
- [ ] Implement Pattern Learning Agent (background, no user interaction)
- [ ] Implement nightly APScheduler job that runs at 02:00 (user's timezone)
- [ ] Job reads: last 7 days of check-ins, task actuals vs estimates, acceptance/override rates
- [ ] Job outputs: updated `user_patterns` table + new Mem0 memories tagged `pattern_learned`
- [ ] Implement time estimation bias calculation: rolling average of (actual_minutes / estimated_minutes)
- [ ] Implement energy pattern extraction: which hour buckets have highest task completion rates
- [ ] Verify: after 7+ days of data, patterns are being correctly extracted

### Week 8: Time Realism + Energy Routing
- [ ] Update Focus Agent to use `time_estimation_bias` when scheduling tasks
- [ ] Update Focus Agent to schedule demanding tasks during user's peak energy window
- [ ] Update Focus Agent to cap daily task count based on `avg_completion_rate_7d`
- [ ] Implement "drift alert": if a task has been deferred 3+ times, Focus Agent surfaces it prominently
- [ ] Test: user estimates 5 tasks at 30 min each. Agent re-estimates using their bias and correctly caps the plan.

### Week 9: Google Calendar Integration
- [ ] Set up Google OAuth (free — Google Calendar API has a generous free tier)
- [ ] Implement `GET /integrations/calendar/connect` — OAuth flow
- [ ] Implement calendar sync: pull events for the next 7 days into PostgreSQL
- [ ] Update Focus Agent to read calendar events before building daily plan
- [ ] Ensure agent never schedules tasks over calendar events
- [ ] Test: add a meeting to Google Calendar, verify it appears in the daily plan

### Week 10: Goals + Relationships Agents
- [ ] Implement Goals Agent
- [ ] Implement `POST /goals` — create goal with milestones
- [ ] Implement `GET /goals` — list goals with progress
- [ ] Implement `PATCH /goals/{id}` — update progress
- [ ] Implement Relationships Agent
- [ ] Implement `POST /relationships` — add important person
- [ ] Implement weekly nudge: if `last_interacted_at > interaction_frequency_days`, surface reminder
- [ ] Add Goals + Relationships to supervisor routing
- [ ] Add Goals + Relationships cards to frontend dashboard

### Week 11: Weekly Review Agent
- [ ] Implement Weekly Review Agent (background, APScheduler Sunday 20:00)
- [ ] Weekly review reads: all check-ins, task history, goal progress, pattern updates for the week
- [ ] Generates personalised summary: wins, struggles, one pattern, one suggested change
- [ ] Stores weekly review in PostgreSQL
- [ ] Implement `GET /reviews/weekly` — returns latest weekly review
- [ ] Add Weekly Review card to frontend dashboard (visible Monday morning)

### Week 12: Refinement + Polish
- [ ] Add "you overrode this suggestion X times" feedback loop — agents learn from overrides
- [ ] Implement manual memory correction: user can view and delete any stored memory
- [ ] Add streak tracking to check-in system
- [ ] Implement "chaos mode" auto-detection: if stress_score ≥ 4 in check-in, auto-trigger triage
- [ ] Performance audit: identify slowest agent calls, switch appropriate ones to Haiku
- [ ] Security audit: verify all routes are protected, no data leaks between users
- [ ] **Phase 2 complete: dog-food for 1 month, gather 5 beta users**

## Phase 3 — Full Swarm + Scale (Months 4–6)
**Goal: Production-ready, all domains covered, mobile app.**

### Architecture upgrade
- [ ] Migrate from Supervisor pattern to LangGraph Swarm (`pip install langgraph-swarm`)
- [ ] Profile routing accuracy first — only migrate if supervisor is a bottleneck
- [ ] Add Finance Agent (spend tracking, bill reminders, savings nudges)
- [ ] Add Delegate Agent with full web research capability

### Mobile
- [ ] Build React Native app (Expo) with same SSE streaming
- [ ] Push notifications for morning check-in, weekly review, chaos alerts

### Data + Monetisation
- [ ] Implement usage metering (track API costs per user)
- [ ] Implement subscription tier (Stripe — free tier + $9/month paid)
- [ ] Free tier: 1 check-in/day, 3 agent requests/day
- [ ] Paid tier: unlimited, all agents, weekly review, memory history
